# 2026-04-06 智能分配策略配置开发记录

## 本次完成

- 新增智能分配策略配置表 `db_consultation_dispatch_config`
- 新增后台管理接口 `/api/admin/consultation-dispatch/config`
- 新增策略预览接口：
  - `GET /api/admin/consultation-dispatch/preview-records`
  - `POST /api/admin/consultation-dispatch/preview`
- 新增后台页面“智能分配策略”，可配置：
  - 接诊方式匹配权重
  - 近期排班权重
  - 号源容量权重
  - 医生负载权重
  - 服务标签权重
  - 标签命中单项分
  - 最多参与加分的匹配标签数
  - 推荐候选医生数
  - 推荐分上下限
  - 待接手超时阈值
- 支持在后台选择真实问诊样本，基于当前页面参数实时预览推荐医生结果，未保存配置也可先看效果
- 推荐医生排序逻辑已改为读取策略配置，不再依赖固定硬编码
- 后台智能分配运营总览中的“超时待接手”阈值已改为读取策略配置

## 当前默认参数

- 各评分因子权重：`100%`
- 标签命中单项分：`4`
- 最多参与加分的匹配标签数：`3`
- 推荐候选医生数：`3`
- 推荐分区间：`24 - 99`
- 待接手超时阈值：`24 小时`
