# 2026-04-06 患者端问诊进度增强记录
## 本轮完成

- 患者问诊列表新增“待医生处理”“待随访”统计。
- 问诊列表新增搜索、状态筛选、进度筛选、随访筛选。
- 问诊列表新增“问诊进度”“随访提醒”两列，并对医生新回复、今日到期随访、已逾期随访做行高亮。
- 问诊详情新增“随访提醒”提示条。
- 问诊详情新增“问诊进度时间线”，串联提交、导诊、智能分配、医生接手、沟通更新、处理完成、随访等关键节点。
- 患者打开详情后会同步刷新列表快照，避免返回列表时进度和随访状态滞后。

## 后端改动

- `ConsultationServiceImpl.listRecords(...)` 现在会一并返回：
  - 最新医生处理记录
  - 最新结构化结论
  - 最近一次随访记录
- 这样患者列表页无需逐条打开详情，也能直接判断：
  - 当前是否已被医生接手
  - 当前是否已完成处理
  - 是否仍需随访以及是否逾期

## 前端改动

- `template-front/src/views/index/ConsultationPage.vue`
  - 增加患者侧问诊列表筛选工具栏。
  - 增加问诊进度与随访提醒列。
  - 增加详情页时间线与随访提醒。
  - 增加列表快照同步逻辑。

## 价值

- 患者无需反复打开详情，也能在列表页直接看懂“医生是否已回复、是否已接手、是否需要继续随访”。
- 患者详情页形成更完整的问诊全过程展示，更符合智能问诊系统的闭环体验。
- 为后续继续做患者侧消息提醒入口、随访催办和状态通知打下基础。
