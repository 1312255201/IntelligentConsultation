# 2026-04-11 管理端服务评价质控视图开发记录

## 本轮目标

在患者已可提交服务评价、医生已可处理评价的基础上，继续补齐管理端视角，让后台可以直接看到：

- 哪些问诊已经有服务评价
- 哪些评价仍需重点关注
- 医生对评价的处理进度如何
- 不同科室的服务评价质量分布情况

- 管理端问诊记录列表接口补充服务评价数据
- 管理端问诊记录页新增“服务评价质控概览”板块
- 新增服务评价统计卡片：
  - 已提交评价
  - 待关注评价
  - 低分评价

- 问诊记录列表新增“服务评价”列
- 问诊记录列表新增服务评价筛选：

  - 待关注评价

  - 医生处理状态

  - 处理医生

## 业务价值

- 后台现在不再只能在详情页被动查看单条服务评价，而是可以直接看到整体质量情况
- “患者评价 -> 医生处理 -> 后台观察”第一次形成了完整管理闭环
- 后续若继续扩展服务评价质控、满意度分析和绩效看板，可以直接复用当前列表数据与筛选口径
