# 2026-04-06 患者端提醒看板开发记录
## 1. 本轮目标

继续沿着“患者提交问诊 -> AI 导诊 -> 医生接手 -> 随访提醒”的主链路推进，把患者端问诊页顶部补成一个可直接使用的提醒看板，让用户进入页面后不用先翻表格，就能快速看到：

- 医生是否有新的回复
- 当前是否仍在等待医生处理
- 哪些问诊已经进入待随访、今日到期或逾期状态

## 2. 本轮完成

### 2.1 患者端提醒看板

在 `template-front/src/views/index/ConsultationPage.vue` 顶部统计卡下方新增了三块提醒卡片：

- `医生新回复`
- `待医生处理`
- `待随访提醒`

每张卡片都会展示当前最需要关注的 4 条问诊记录，并提供对应的快捷入口。

### 2.2 快速筛选联动

提醒卡片底部补充了“只看医生新回复”“只看待处理问诊”“只看待随访”“优先看逾期”等快捷按钮。

点击后会自动：

- 同步切换问诊记录区的筛选条件
- 平滑滚动到历史记录区
- 直接进入当前提醒场景下的列表视图

### 2.3 筛选状态可恢复

为了让提醒入口更稳定，本轮把患者问诊页的列表筛选条件同步到了路由查询参数中，包括：

- 关键词
- 状态
- 问诊进度
- 随访状态

这样做之后：

- 用户从提醒卡片进入历史记录区后刷新页面，筛选状态不会丢
- 后续如果要从其他页面带条件跳转到患者问诊页，可以直接复用这套查询参数

### 2.4 响应式补齐

补充了提醒看板在中小屏下的布局适配：

- `1180px` 以下三列提醒卡自动折叠为单列
- `760px` 以下提醒头部、条目头部、标签区和底部操作区自动纵向排列

这样可以避免提醒卡在窄屏下出现拥挤、换行混乱的问题。

## 4. 价值

这一轮虽然没有新增复杂业务表，但把患者端“提醒感知”补了出来，实际意义很大：

- 用户进入页面后能先看到待办，而不是只看到历史表格
- 患者更容易感知医生是否已回复、是否仍在等待处理、是否需要继续随访
- 为后续继续做患者消息提醒入口、通知中心、状态催办和跨页面条件跳转打下了前端结构基础
