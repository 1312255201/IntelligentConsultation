# 2026-04-06 患者工作台首页升级记录

## 1. 本轮目标

把原先偏“系统概览”的患者首页改造成真正可用的患者工作台，让用户进入患者端后，第一眼就能看到：

- 当前有哪些待办提醒
- 最近问诊推进到了哪一步
- 健康档案覆盖情况是否完整
- 常用入口是否能直接跳到问诊、提醒和 AI 导诊

## 2. 本轮完成

### 2.1 患者首页升级为工作台

更新页面：

- `template-front/src/views/index/OverviewPage.vue`

首页不再只是静态概览，而是围绕患者侧全流程做成了一个工作台入口，包含：

- 顶部工作台摘要区
  - 汇总当前提醒数量
  - 根据逾期随访、医生新回复、待医生处理等状态生成首页提示文案
- 核心统计区
  - 就诊人数
  - 健康档案覆盖数
  - 问诊总数
  - 医生新回复数
  - 待医生处理数
  - 待随访、今日到期、逾期随访数量
- 当前待办区
  - 优先展示最值得先处理的提醒记录
  - 支持直接跳转到对应问诊详情
- 快捷入口区
  - 发起问诊
  - 消息与提醒
  - AI 导诊工作区
  - 健康档案
  - 就诊人管理
  - 账户设置
- 最近问诊区
  - 展示最近推进中的问诊记录
  - 可直接回到问诊详情或 AI 导诊工作区
- 账户与档案摘要区
  - 展示当前账号信息
  - 展示健康档案覆盖率和问诊完成占比

### 2.2 患者端默认入口切换到工作台

更新文件：

- `template-front/src/router/index.js`
- `template-front/src/views/IndexView.vue`

本轮把患者工作区默认重定向从 `/index/profile` 调整为 `/index/overview`，同时把左侧菜单重排为“患者工作台”优先，这样用户进入患者端后会先落到真正的业务首页，而不是资料页。

## 4. 价值

这次调整之后，患者侧终于形成了明确的首页定位：

- 首页变成患者当前状态的总入口，而不是单纯的展示页
- 用户进入系统后更容易先看到提醒和待办
- 后续继续接入通知、随访催办、智能推荐时，有稳定的首页承接位置
