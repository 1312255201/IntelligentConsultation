# 2026-04-06 患者端消息与提醒中心开发记录
## 1. 本轮目标

在已经完成患者端问诊列表提醒看板的基础上，继续把“提醒能力”从单页顶部卡片扩展为独立入口，让患者可以直接进入一个专门的消息与提醒中心，集中处理：

- 医生新回复
- 待医生处理的问诊
- 待随访、今日到期和已逾期的问诊

同时补齐跨页面跳转能力，让提醒中心可以直接把用户带到某条问诊详情，而不是只跳到列表页。

## 2. 本轮完成

### 2.1 新增患者端独立提醒中心页面

新增页面：

- `template-front/src/views/index/ConsultationReminderPage.vue`

页面提供了三层信息组织：

- 顶部总览区
  - 展示总提醒数、医生新回复数、待医生处理数、待随访数、今日到期数、逾期数
  - 根据当前最紧急事项生成一段摘要文案
- 中部优先提醒卡片
  - `医生新回复`
  - `待医生处理`
  - `待随访提醒`
- 底部提醒流
  - 支持按 `全部 / 医生回复 / 待处理 / 待随访 / 逾期` 切换查看
  - 每条提醒都能直接进入对应问诊或 AI 导诊工作区

### 2.2 新增患者工作区菜单入口

更新了患者工作区左侧导航与路由：

- 新增菜单 `消息与提醒`
- 新增路由 `/index/reminder`

这样患者不需要先进“发起问诊”页，也能直接进入提醒中心。

### 2.3 问诊页支持按路由参数直接打开详情

为了让提醒中心能真正形成闭环，本轮补充了患者问诊页的详情深链能力：

- `template-front/src/views/index/ConsultationPage.vue`

新增能力包括：

- 支持通过路由参数 `id` 自动打开指定问诊详情
- 从外部页面跳转进来时，可保留原有筛选条件并直接打开详情
- 关闭详情后自动移除 `id` 参数，避免页面状态残留
- 列表重新加载后，如果路由中仍带 `id`，会自动尝试恢复详情弹窗

这样后续不只是提醒中心，首页提醒、通知中心、消息入口等页面也都可以复用同一套跳转方式。
