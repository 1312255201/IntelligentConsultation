# 2026-04-11 医生端服务评价待办闭环开发记录
## 本轮目标

在患者已能提交服务评价、医生工作台已能看到评价概览的基础上，继续补齐医生侧“提醒中心 -> 问诊列表 -> 问诊详情”的闭环，让医生能直接筛出、打开并定位到需要重点处理的服务评价。

## 本轮完成

- 医生问诊页新增“待关注评价”统计卡
- 医生问诊列表新增服务评价筛选器
- 医生问诊列表新增服务评价摘要列
- 医生问诊页支持 `feedbackFilter` 路由参数同步
- 医生问诊页支持 `action=feedback`，可从提醒中心直接定位到服务评价区块
- 医生问诊详情中的服务评价面板增加聚焦高亮
- 医生提醒中心顶部摘要与统计卡补齐“待关注评价”展示

## 前端改动

更新文件：

- `template-front/src/views/doctor/DoctorConsultationPage.vue`
- `template-front/src/views/doctor/DoctorReminderPage.vue`

实现说明：

- 问诊列表新增 `feedbackFilter`，支持：
  - `attention`
  - `has_feedback`
  - `low_score`
  - `unresolved`
- 问诊列表中的服务评价列会展示：
  - 评价状态标签
  - 评分
  - 是否解决
  - 评价时间
  - 评价摘要
- 来自提醒中心的 `action=feedback` 会在打开详情后自动滚动到服务评价区域，并做短暂高亮
- 问诊列表会对低分或未解决评价增加行级提醒色，方便医生快速识别
- 提醒中心顶部 hero 摘要会优先提示“待关注服务评价”

## 业务价值

- 医生不再只能在详情页被动看到服务评价，而是可以主动筛出重点反馈
- 低分与未解决评价具备了统一入口，后续更容易衔接回访、备注和质控
- 服务评价首次真正进入医生日常待办流，而不是只作为历史信息沉淀

## 下一步建议

- 在服务评价详情区补充“处理备注 / 回访记录”能力
- 在医生工作台增加“待关注评价”快捷入口按钮
- 在管理端补充按医生、科室聚合的服务评价质量分析
