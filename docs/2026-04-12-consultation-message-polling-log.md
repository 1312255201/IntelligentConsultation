# 2026-04-12 问诊消息自动刷新开发记录

## 本次完成

- 患者端问诊详情页增加消息自动轮询。
- 医生端问诊详情抽屉增加消息自动轮询。
- 轮询仅在问诊详情打开且页面处于可见状态时运行，避免后台标签页无意义请求。
- 页面切到后台时自动停止轮询，切回前台后自动恢复并立即同步一次消息。
- 消息刷新支持静默模式，自动轮询时不会反复出现加载闪烁。
- 消息列表增加滚动位置保护：
  - 初次打开详情会自动滚动到最新消息。
  - 如果当前已经在底部，自动刷新后会继续保持在最新消息位置。
  - 如果用户正在上翻查看旧消息，自动刷新不会强行拉到底部。

## 涉及页面

- `template-front/src/views/index/ConsultationPage.vue`
- `template-front/src/views/doctor/DoctorConsultationPage.vue`

## 验证

- 前端构建通过：
  - `npm run build`

## 对在线问诊的价值

- 医患双方打开问诊详情后，不需要频繁手动点击“刷新消息”。
- 在线问诊体验更接近即时沟通，医生回复和患者补充可以更快被看到。
- 现有未读状态与消息摘要也会随着自动同步及时更新。

## 后续建议

1. 增加“最近同步时间”或“在线同步中”轻提示，让用户知道页面正在自动刷新。
2. 将当前轮询进一步升级为 WebSocket 长连接，减少无效请求。
3. 在有新消息且用户不在底部时，增加“收到新消息，点击跳转到底部”的提示条。
